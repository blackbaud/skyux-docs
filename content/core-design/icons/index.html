Icons

Icons are pictorial representations of objects, important not only for aesthetic reasons as part of the visual identity of a program, but also for utilitarian reasons as shorthand for conveying meaning that users perceive almost instantaneously. {{ stache.config.product_name_short }} uses glyphic mono-color icons to facilitate, guide and inform users.

Usage

Context for usage

  • Provide quick recognition of common actions and tasks.
  • Quickly give added information to a record.
  • Denote type or status.

Icon selection

  • If you need an icon for a new component, determine if it is currently recommended in {{ stache.config.product_name_short }}, and consider where and how to make sure that the meaning you prescribe to it is the same.
  • Don't repurpose an established icon for a different action.
  • Be mindful of how icons are used in specific components, like list and tile toolbars, buttons, and forms.
  • If an icon does not add to or facilitate the user experience, then it is best not to use one.

Styling

Icons will usually inherit their size and color from their parent element, particularly when they are used inline. If you are using icons in a non-inline context and need a different size, FontAwesome provides size multiplier classes. Use the smallest size multiple that accomplishes your goals. If you are applying a different color to an icon, be sure if follows the usage as described in the Color documentation.
