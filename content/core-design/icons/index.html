Icons

Icons are pictorial representations of objects. They help define the visual identity of {{ stache.config.product_name_short }} applications, and they provide a shorthand to visually convey meaning that users can unerstand at a glance. {{ stache.config.product_name_short }} uses glyphic, mono-color icons to facilitate, guide, and inform users.

Usage

Context for usage

  • Provide visual cues for quick recognition of common actions and tasks
  • Quickly give added information to a record
  • Denote type or status

Icon selection

  • To select an icon for a new component, check the icon dictionary on this page to determine if it is available in {{ stache.config.product_name_short }}. Then make sure that your use case is consistent with the way it is used in other {{ stache.config.product_name_short }} applications.
  • Don't use an icon with an established meaning for a different purpose.
  • Be mindful of how icons are used in specific components, like list and tile toolbars, buttons, and forms.
  • Do not use an icon if it does not add to or facilitate the user experience.

Styling

Icons usually inherit their size and color from parent elements, particularly when you use them inline. If you use icons in a non-inline context and need a different size, FontAwesome provides size multiplier classes. Use the smallest size multiple that accomplishes your goals. To are apply a different color to an icon, follow the Color guidelines.
